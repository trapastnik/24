# МТК №24 «Ленин в октябре» — проектные инструкции

Кинематографичная видео-проекция на three.js: историческая карта Петрограда как
«земля», камера с наездами по сценарию, маркеры сил / маршруты / HUD-титры.
Запуск — статикой: `index.html` (importmap → `lib/three`, данные из `data/*.js`,
логика в `scene.js`).

## ⚠️ ПАРАЛЛЕЛЬНЫЕ СЕССИИ — РАЗДЕЛЕНИЕ ВЛАДЕНИЯ ФАЙЛАМИ

Над проектом одновременно работают несколько сессий Claude Code. Чтобы правки не
затирали друг друга, действует жёсткий контракт владения. **Редактируй только
свои файлы. Если нужен чужой файл — не правь его, а передай запрос через главный
(оркестрирующий) чат.**

### Сессия «генерация моделей» владеет:
- `assets/models/**` — выходные `.glb`
- `tools/build_models.mjs` — генератор моделей
- `tools/extract_shape.py` — извлечение контуров (пишет только в `/tmp/`)
- `data/models.js` — конфиг моделей (`window.MTK24_MODELS`: `dir`, `cfg{file,size,yaw}`).
  Подбор `size`/`yaw` каждой модели делается ЗДЕСЬ, а не в `scene.js`.

### Основная сессия («всё важное») владеет:
- `scene.js` — вся логика сцены. **MODEL_CFG больше не живёт здесь**: scene.js
  только читает `window.MTK24_MODELS` из `data/models.js` (не редактирует его).
- `styles.css`, `index.html`
- `data/locations.js`, `data/scenario.js`, `data/streets.js`
- остальные `tools/*.py` (редактор/гереференс/OSM)

### Никто не трогает без согласования:
- `lib/three/**` — вендор three.js
- `assets/map/**`, `fonts/**`, `IN/**`, `data/*.bak`

## Git-дисциплина (во избежание взаимного затирания)
- **Не делать `git add -A` / `git commit -am`** — заметёт чужую недоделанную работу.
  Коммить ТОЛЬКО свои файлы поимённо (`git add scene.js styles.css …`).
- **Не делать** `git checkout -- <file>`, `git reset --hard`, `git stash` по общим
  файлам без согласования через главный чат.
- Коммиты и пуш — по запросу пользователя, не самовольно.

## Деплой — ТОЛЬКО через GitHub (НЕ напрямую!)
**Запрещён прямой деплой** (rsync/scp с Mac на сервер). Деплой идёт только через git.
Порядок:
1. Локально: убедиться, что **все коммиты со всех веток** подтянуты —
   `git fetch --all`, слить нужное в `main`. Это ОБЯЗАТЕЛЬНО, чтобы не потерять
   работу параллельных сессий/веток.
2. `git push origin main` (репозиторий публичный: github.com/trapastnik/24).
3. На сервере `/var/www/mtk24` (это git-checkout того же репо):
   `git fetch --all && git reset --hard origin/main` → затем `chmod -R a+rX /var/www/mtk24`.
   Контейнер `mtk24-web` подхватывает файлы сам (bind-маунт), рестарт не нужен.
- `nginx.conf` и `docker-compose.yml` на сервере НЕ в репозитории (untracked) —
  git их не трогает.

## Контракт интерфейса «модели ↔ сцена»
`data/models.js` подключается в `index.html` обычным `<script>` ДО модуля
`scene.js` и выставляет `window.MTK24_MODELS = { dir, cfg }`. `scene.js` читает его
в начале (как `MTK24_LOCATIONS` / `MTK24_SCENARIO`). Ключи `cfg` должны совпадать с
ключами точек в `data/locations.js` (`smolny`, `winter`, `fortress`, `mariinsky`,
`tauride`, `aurora`).
