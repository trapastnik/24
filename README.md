# МТК №24 · «Ленин в октябре»

Кинематографичная **видео-проекция «24–25 октября 1917»** для музея В.И. Ленина:
объёмная карта Петрограда, по которой в хронологии разворачивается вооружённое
восстание — от подполья к Совнаркому. По сценарию каждый кадр — точки/объекты на
карте, которые загораются и меняют цвет; движение отрядов — по улицам.

Технология: **three.js (WebGL)** + vanilla JS, без бандлеров. Брендовая палитра
музея (paper / brass / signal-red / graphite), шрифты Nolde / 20 Kopeek.

> Репозиторий: https://github.com/trapastnik/24

## Запуск

```bash
cd BMK/24
python3 tools/edit_server.py          # → http://localhost:8124/
```
- **Ролик:** http://localhost:8124/index.html — пробел: пуск/пауза, ←/→: по кадрам, клик по таймлайну: перемотка.
- **Редактор точек/маршрутов/улиц:** http://localhost:8124/tools/authoring.html

`edit_server.py` отдаёт статику и принимает сохранения из редактора
(`POST /api/save-locations`, `/api/save-streets`). Обычный `python3 -m http.server`
тоже покажет ролик, но без сохранения из редактора.

## Что на экране

- **Рабочий экран** — точная форма проекции по ТЗ (`in/размер 24.PNG`): трапеция с
  гладкими дугами сверху/снизу (`shape-clip.js`, `clip-path: path()`). Всё событийное
  происходит внутри него.
- **Объёмная карта** Петрограда (исторический «ПЛАНЪ ПЕТРОГРАДА») как наклонная
  «земля»; кинематографичная камера ведётся по кадрам (без вращения).
- **3D-модели** ориентиров (Смольный, Зимний, Петропавловка, Мариинский, Таврический,
  «Аврора») + объёмы-боксы для прочих объектов. Цвет силы: **ВРК — жёлтый+звезда**,
  **Временное правительство — чёрный**, **захвачено — красный**.
- **Маршрут Ленина** (7 этапов) и переброски — анимация по улицам.
- **Часы + календарь** по реальному времени события (идут 24 → 26 октября).
- **Таймлайн** снизу — по дате/времени события (не по минутам видео).
- **ГЗК (правая панель)** — технический «лист кадра»: номер кадра, заголовок, время
  события, тайминг видео, эффект, объекты, иллюстрация + ПОЛНЫЙ дикторский текст.
- Титры, цитаты, фото-врезки.

11 сцен, хронометраж ≈ 4:17.

## Структура

```
index.html        раскладка: рабочий экран + правая ГЗК-панель + нижний таймлайн
scene.js          three.js-сцена: карта, камера, модели/объекты, маршруты, HUD, часы, техпанель
shape-clip.js     обрезка рабочего экрана по точной форме ТЗ (дуги)
styles.css        палитра, шрифты, панели

data/scenario.js  СЦЕНАРИЙ (редактируемый): 11 кадров — тайминг видео (t0/t1) и времени
                  события (s0/s1), narration, voFull (полный текст), цитаты, иллюстрации,
                  точки/маршруты/эффекты
data/locations.js точки (нормированные [u,v] по карте) + маршруты (7 этапов Ленина и др.)
data/streets.js   подписи улиц (плашки): позиция + угол
data/models.js    конфиг 3D-моделей (window.MTK24_MODELS)        ← сессия моделей
data/petrograd_osm.{json,js}  геометрия центра из OSM (для привязки/маршрутов)

assets/map/       текстуры карты (jpg для веба)
assets/ill/       иллюстрации-врезки (кадры)
assets/models/*.glb  лёгкие low-poly модели ориентиров              ← сессия моделей
lib/three/        three.js + addons (GLTFLoader, RoomEnvironment …)

tools/
  authoring.html       визуальный редактор точек/маршрутов/улиц (зум, перетаскивание,
                       поворот, удаление, автосохранение, Ctrl+Z)
  edit_server.py       статика + запись data/locations.js, data/streets.js
  fetch_osm.py         геометрия центра из Overpass
  place_from_pdf.py    позиции точек из ВЕКТОРНОГО PDF (пины→гомография→uv)
  extract_streets.py   улицы/реки из текста PDF → data/streets.js
  extract_shape.py     точная форма экрана из in/размер 24.PNG → shape-clip.js
  build_models.mjs     генератор GLB-моделей                          ← сессия моделей

prototype-v1.html + map24.js    архив первого интерактивного 2D-прототипа
```

## Данные и привязка

Готовой векторной карты восстания нет — карта собрана дизайнером (исходники в `in/`,
не в репозитории из-за размера). Точки привязаны к карте из **векторного PDF**
(`tools/place_from_pdf.py`: оранжевые пины ↔ подписи, гомография в систему чистой
карты). Координаты — нормированные `[u,v]`; правятся в `tools/authoring.html`.
