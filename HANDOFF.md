# HANDOFF — МТК №24 «Ленин в октябре»

Состояние проекта для продолжения в новом чате. Кратко: рабочая кинематографичная
видео-проекция на three.js, 11 сцен (≈4:17). Подробности — в [README.md](README.md).

## Запуск / проверка
```bash
cd ~/Desktop/WWWWW/BMK/24
python3 tools/edit_server.py        # → http://localhost:8124/index.html  (пробел = пуск)
```
Превью: открыть `index.html`. Редактор: `tools/authoring.html`. Репозиторий:
https://github.com/trapastnik/24 (ветка `main`).

## Сделано
- Привязка карты: из векторного PDF (`place_from_pdf.py`) — все точки на местах.
- Сценарий `data/scenario.js`: 11 сцен, тайминг видео (t0/t1) + время события (s0/s1),
  полный дикторский текст (`voFull`), цитаты, иллюстрации, точки/маршруты/эффекты.
- Форма рабочего экрана — точно по ТЗ (`shape-clip.js`, гладкие дуги, масштабируется).
- Часы+календарь по времени события; таймлайн по дате/времени.
- 3D-модели ориентиров (сессия моделей) + объёмы-боксы; цвета сил, смена при захвате.
- **Объекты ПОСТОЯННЫЕ** (`scene.js`: `buildObjects/applyShotToObjects/updateObjects`):
  строятся один раз и живут весь ролик. Не в фокусе кадра — полупрозрачные (приглушены),
  в фокусе — полная яркость + цвет силы + пульс. «Захват» (`becomes:red`) монотонен по
  кадрам (раз красный — остаётся, корректно при перемотке). НЕ перестраивать покадрово.
  Любой ориентир с моделью из `data/models.js` строится как постоянный фоновый объект,
  даже если его нет ни в одном кадре сценария (напр. Таврический): сила — из
  `locations.js`, он всегда приглушён. Чтобы такой объект стал ярким в каком-то кадре —
  добавить его ключ в `points[]` этого кадра (зона основной сессии, `data/scenario.js`).
- Маршрут Ленина = 7 этапов + промежуточные; редактор `authoring.html` (перетаскивание,
  поворот, удаление, автосейв, Ctrl+Z), слой улиц-плашек.
- Раскладка: ГЗК (полный текст) — правая панель; таймлайн — снизу; титры не режутся формой.
- Техпанель: номер кадра + метаданные постановки (событие/видео/эффект/объекты) + ГЗК.
- Тайминг: Кадр 3 «Телеграф» ускорен 1.5× (27с), Кадр 4 «Ленин» — 20с.
- **Спецэффекты кадров (`fx[]`) реализованы в `scene.js`** (блок «FX»): пунктир
  телеграмм, волна от Смольного, сжимающееся кольцо у Зимнего, выстрел «Авроры»
  (сигнал→дульная вспышка→ударная волна→снаряд→удар), заливка красным «как чернила»,
  лучи Смольного + полноэкранная вспышка (HUD-оверлей `#fx-flash`). Каждый эффект
  знает свой `at` и считает фазу `fp = (lp − at)/(1 − at)`; диспетчер `FX_BUILD[type]`,
  сборка `buildFx(shot)`, анимация `updateFx(lp, time)`. Все 7 типов проверены в превью.

## Осталось (зона сцены/UI)
1. **Базовые звуки** — отдельный `audio.js` (свой файл): фон + холостой выстрел + щелчки,
   синхронно с таймлайном (WebAudio, без внешних файлов; озвучка диктора — слотами).
2. **Живой редактор крупности/положения камеры** — пауза + правка кадрирования прямо
   в ролике, сохранение покадрово (в `scenario.js` поле camera/framing).
3. **Подсветка целых улиц** (не только названия) — слой по геометрии улиц (OSM-роуды
   уже есть в `data/petrograd_osm.json`).

## Технические заметки
- Координаты точек/маршрутов — нормированные `[u,v]` по изображению карты;
  `scene.js: uvToWorld(u,v)` → плоскость. Маршрут: точки бывают `[u,v]` (промежуточные)
  и `{u,v,n,name}` (этапы с номером).
- Время события: `s0/s1` — минуты от 24 окт 00:00 (25 окт = +1440, 26 окт = +2880);
  `scene.js: storyDT(min)` форматирует дату/время. Часы интерполируют s0→s1 по прогрессу.
- Форма экрана: углы/прогиб дуг из `tools/extract_shape.py` (in/размер 24.PNG, аспект ≈1.147),
  применяется как `clip-path: path()` в `shape-clip.js` (ResizeObserver).
- Титр кадра делает плавный кросс-фейд (~0.25с) — на скриншотах может «отставать»,
  вживую синхронен (не баг).
- `in/` (исходные PSD/PDF/фото, ~0.5 ГБ) — НЕ в репозитории (.gitignore); нужны только
  для перегенерации (tools/*.py). Веб-ассеты (texture jpg, ill, glb) закоммичены.
- three.js здесь ревизии r123 — аддоны должны совпадать по версии.
